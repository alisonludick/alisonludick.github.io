What coding language will you choose

HTML and CSS? JavaScript? There are so many languages to choose from. It was helpful when I was seeking advice, to have it explained which languages serve which purpose. I learned that HTML, CSS and JavaScript are used in Front-End development, and that there were other languages used for Back-End. Therefore, deciding which language to study depends, largely, on what kind of a developer you want to become.

How to prepare your machine

What operating system do I use? And what version? What text editor should I use? I had no idea was a text editor even was but I have a friend who is a senior software engineer, who sent me a very helpful email giving advice about these things. He suggested Visual Studio Code, so that's the one I went for. Never underestimate the value of asking those who have the relevant experience and know-how. I discovered that, because I have a Macbook with Yosemite, version 10.10.5, I needed to download the appropriate version of VSCode. I have since had it automatically updated so now I'm working on Version 1.21.0-insider, and it is working well for me! I was also advised to download Git and, again, because of my version of Mac, I had to download an older version. The error message I got when I installed the latest version said "no mountable file systems" (no idea?!) and I had to investigate what that meant. I realised I needed an older version of Git. It took me quite a while to figure this out but I eventually found what I needed here: SourceForge. There were also some helpful folks in the GitHub community who shared this same source. That wasn't the end of it. As I started taking some modules on HTML and CSS at edX, I was unable to run something called Sass and, again, I reached out to the coding community on social media (FYI: Twitter is an excellent place to get your questions answered!), and someone was kind enough to advise that I needed something called Homebrew to run Sass. He said: "Apple has a nasty habit of not installing Dev headers for things." He said I should install Homebrew and then install its version of Ruby and that would enable me to run Sass. He added: "This will then put an up to date copy of Ruby in /usr/local/bin/ruby and add to your path. This copy has the missing headers available." Can I just say how grateful I am for people who know more than I do, who are willing to share their knowledge? Yikes! I still don't know what he meant by all that, but I am sure am thankful that I have been able to proceed with learning code because of the help of others.
Update, Feb 11: After using Brew to install Ruby, I got this message: files have been installed to /usr/local/Cellar/ruby/2.5.0 and Emacs Lisp files have been installed to /usr/local/share/emacs/site-lisp/ruby.
